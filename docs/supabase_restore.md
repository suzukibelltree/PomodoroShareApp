## Suggested restore commands

If PostgreSQL tools are available locally, the backup is likely intended for `pg_restore`.
You will need the new project's database host, database name, user, and password from Supabase.

Example:

```powershell
pg_restore `
  --verbose `
  --clean `
  --if-exists `
  --no-owner `
  --no-privileges `
  --dbname="postgresql://postgres:<PASSWORD>@db.<PROJECT-REF>.supabase.co:5432/postgres?sslmode=require" `
  "C:\Users\a7p7p\Downloads\db_cluster-06-10-2025@16-14-22.backup"
```

Replace:

- `<PASSWORD>` with the new project database password
- `<PROJECT-REF>` with `pjukeyzyhhrlhabfrnvx`

## Suggested storage restore

A restore helper script has been added at:

- `scripts/restore_supabase_storage.ps1`

It uploads the local storage export back into the new Supabase Storage bucket while preserving object keys.

Example:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\restore_supabase_storage.ps1 `
  -ProjectUrl "https://pjukeyzyhhrlhabfrnvx.supabase.co" `
  -ServiceRoleKey "<SERVICE_ROLE_KEY>" `
  -SourceRoot "C:\Users\a7p7p\Downloads\jaemimxpboicrxbpaycq.storage\jaemimxpboicrxbpaycq\pomodoro" `
  -BucketName "pomodoro" `
  -CreateBucketIfMissing
```

Use the Supabase `service_role` key for this script so it can upload objects regardless of bucket policy.

- Edge Functions
